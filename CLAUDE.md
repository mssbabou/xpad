## Project Overview

xpad is a Flutter dashboard app for a Raspberry Pi touchscreen. It shows weather, air quality, and printer status. It also contains native C code linked via Dart FFI, compiled at build time via `hooks` + `native_toolchain_c`, with bindings generated by `ffigen`.

## App Structure

### Pages (4 swipeable tabs in `lib/pages/home_page.dart`)

1. **Dashboard** (`lib/pages/dashboard_page.dart`) ‚Äî Clock, current weather snippet, mustiness (air quality) gauge
2. **Climate** (`lib/pages/climate_page.dart`) ‚Äî Indoor/pollen block, hourly weather forecast
3. **OctoPrint** (`lib/pages/octoprint_page.dart`) ‚Äî 3D printer status, file list, temperatures, preheat
4. **Settings** (`lib/pages/settings_page.dart`) ‚Äî Routes to 4 sub-pages:
   - **General** (`lib/pages/settings/general_settings_page.dart`) ‚Äî Manual location override
   - **Wi-Fi** (`lib/pages/settings/wifi_settings_page.dart`) ‚Äî Adapter toggle, network scan
   - **OctoPrint** (`lib/pages/settings/octoprint_settings_page.dart`) ‚Äî URL + API key
   - **Debug** (`lib/pages/settings/debug_settings_page.dart`) ‚Äî System info, data refresh, quit/reboot/shutdown

### Global state (`lib/app/app_state.dart`)

Service singletons accessed directly by pages (no DI framework):

## Services

Pattern: `lib/services/<name>/` with `_api.dart` (HTTP), `_models.dart` (data), `_service.dart` (cache + streams). Services re-export `lib/core/result.dart` and their own models so callers only need one import.

| Service | Description |
|---------|-------------|
| `WeatherService` | Open-Meteo API, 10-min in-memory cache, `weatherStream()` every 15 min |
| `AirQualityService` | Open-Meteo air quality API, 30-min in-memory cache |
| `LocationService` | IP-based geolocation (ip-api.com) with disk persistence via SharedPreferences; supports manual override stored under `location_manual_override` key |
| `GeocodingApi` | Open-Meteo geocoding ‚Äî converts city+country to lat/lon; used by General settings |
| `OctoPrintService` | OctoPrint REST API; config (URL + API key) persisted to SharedPreferences |
| `SystemService` | Reads `/proc` and `/sys` for CPU temp, load, RAM, uptime (Linux only) |
| `WifiService` | Wraps `nmcli` for adapter control and network scanning (Linux only) |

Both `WeatherService` and `AirQualityService` expose `updateLocation(lat, lon)` to hot-swap coordinates without restarting the app.

### Shared extension

`lib/widgets/weather_condition_icon.dart` ‚Äî `WeatherConditionIcon` extension on `WeatherCondition` maps each condition to a Material icon. Import this instead of redefining the mapping per-page.

## Core types

- **`lib/core/result.dart`** ‚Äî Sealed `Result<T>` (`Success` / `Failure`) with `.when()`. `AppError` carries `ErrorKind`, a user-safe `message`, and optional `debugDetail`.

## Native (C ‚Üí FFI)

- **`native/header/xpad.h`** ‚Äî C header declaring the native API
- **`native/src/xpad.c`** ‚Äî C implementation
- **`lib/xpad.g.dart`** ‚Äî Auto-generated FFI bindings (do not edit manually)
- **`tool/ffigen.dart`** ‚Äî Regenerates `lib/xpad.g.dart` from the C header
- **`hook/build.dart`** ‚Äî Compiles C sources at build time via `native_toolchain_c`

`hook/build.dart` auto-discovers all `.c` files in `native/src/` ‚Äî just add new files there. After modifying `native/header/xpad.h`, run `dart run tool/ffigen.dart` to regenerate bindings.

## Key Commands

```bash
dart run tool/ffigen.dart   # regenerate FFI bindings after C header changes
flutter run                 # build and run (C compiled automatically)
flutter analyze             # static analysis
flutter test                # run tests
```
